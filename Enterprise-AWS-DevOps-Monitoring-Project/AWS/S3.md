# Amazon S3 Configuration

## Objective

To create centralized object storage for application artifacts,
logs, backups, and deployment files.

---

## Bucket Details

| Parameter | Value |
|-----------|--------|
| Bucket Name | abc-healthcare-storage-bharath2026 |
| Region | ap-south-1 |

---

## Bucket Structure

```text
application/
logs/
backup/
```

---

## Upload Validation

Created test file:

```bash
echo "test" > test.txt
```

Uploaded:

```bash
aws s3 cp test.txt \
s3://abc-healthcare-storage-bharath2026/application/
```

Verified:

```bash
aws s3 ls \
s3://abc-healthcare-storage-bharath2026/application/
```

---

## Why S3?

- Highly durable
- Scalable
- Secure
- Cost effective
- Native AWS integration

---

## Security

Access controlled through:

- IAM Roles
- IAM Policies

No access keys stored on EC2.
