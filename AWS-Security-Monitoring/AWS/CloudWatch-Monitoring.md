# 📊 Amazon CloudWatch Monitoring

## 📌 Overview

Amazon CloudWatch is a fully managed monitoring and observability service that enables organizations to collect logs, monitor metrics, create alarms, and automate responses for AWS resources and applications.

In this project, Amazon CloudWatch was used to build a **real-time security monitoring pipeline**. CloudTrail logs were streamed into CloudWatch Logs, Metric Filters identified failed AWS Console login events, and CloudWatch Alarms generated alerts that were delivered through Amazon SNS.

---

# 🎯 Objective

The objective of implementing Amazon CloudWatch was to:

- Monitor CloudTrail logs in near real time.
- Detect failed AWS Console login attempts.
- Convert log events into CloudWatch metrics.
- Generate automated alarms.
- Notify administrators through Amazon SNS.
- Improve cloud security monitoring and incident response.

---

# 🏗️ Services Implemented

This implementation consists of three CloudWatch components:

- Amazon CloudWatch Logs
- CloudWatch Metric Filters
- CloudWatch Alarms

Together they provide an automated monitoring and alerting solution.

# 1️⃣ Amazon CloudWatch Logs

## What is CloudWatch Logs?

CloudWatch Logs is a centralized log management service that stores logs generated by AWS services and applications.

It enables:

- Log storage
- Log analysis
- Log searching
- Real-time monitoring
- Security event detection

---

## Why We Used CloudWatch Logs

Although CloudTrail stores logs in Amazon S3, S3 is designed for long-term storage and not for real-time analysis.

CloudWatch Logs allows security events to be processed immediately without waiting for log files to be delivered to S3.

---

## What We Implemented

### Step 1

Created a CloudWatch Log Group.

Example

```
CloudTrailLogs
```

Purpose

Store CloudTrail events in near real time.

---

### Step 2

Configured AWS CloudTrail to stream logs into CloudWatch Logs.

Purpose

Enable continuous monitoring of AWS account activities.

---

### Step 3

Verified CloudTrail events inside CloudWatch Logs.

Examples

- ConsoleLogin
- CreateBucket
- RunInstances
- CreateSecurityGroup
- PutMetricAlarm

## Benefits

# 2️⃣ CloudWatch Metric Filter

## What is a Metric Filter?

A Metric Filter scans CloudWatch Logs for specific patterns and converts matching log entries into numerical CloudWatch metrics.

CloudWatch Alarms can only monitor metrics—not raw log data.

Metric Filters bridge this gap.

---

## Why We Used Metric Filters

Our goal was to detect failed AWS Console login attempts automatically.

Whenever CloudTrail generated a failed login event,

the Metric Filter converted it into a custom CloudWatch metric.

---

## What We Implemented

### Metric Filter Name

```
FailedConsoleLoginCount
```

---

### Namespace

```
SecurityMonitoring
```

---

### Metric Name

```
FailedConsoleLoginCount
```

---

### Metric Value

```
1
```

---

### Filter Pattern

```text
{ ($.eventName = "ConsoleLogin") && ($.responseElements.ConsoleLogin = "Failure") }
```

---

## How It Works

Whenever CloudWatch Logs receives a log containing:

```
ConsoleLogin = Failure
```

The Metric Filter increments:

```
FailedConsoleLoginCount
```

This creates a custom CloudWatch metric.

---

## Benefits

- Detects specific log events
- Converts logs into metrics
- Enables automated monitoring
- Supports CloudWatch Alarms

# 3️⃣ CloudWatch Alarm

## What is a CloudWatch Alarm?

A CloudWatch Alarm continuously monitors CloudWatch metrics.

When a predefined threshold is reached,

the alarm changes its state and can trigger automated actions.

---

## Why We Used CloudWatch Alarm

The objective was to notify administrators whenever a failed AWS Console login occurred.

Instead of manually checking logs,

CloudWatch automatically generated an alert.

---

## What We Implemented

### Alarm Name

```
FailedConsoleLoginAlarm
```

---

### Metric

```
FailedConsoleLoginCount
```

---

### Condition

```
Threshold >= 1
```

---

### Evaluation Period

```
5 Minutes
```

---

### Alarm Action

```
Publish notification to Amazon SNS
```

---

## Alarm States

CloudWatch Alarm can exist in three states.

### OK

No failed login detected.

---

### ALARM

At least one failed login detected.

Amazon SNS sends an email notification.

---

### INSUFFICIENT_DATA

CloudWatch does not have enough recent metric data.

This state is common after the monitoring window expires.

# Interview Questions

## Why did you integrate CloudTrail with CloudWatch Logs?

CloudTrail stores logs for auditing, while CloudWatch Logs enables near real-time log analysis. This integration allows Metric Filters to detect security events immediately.

---

## Why are Metric Filters required?

CloudWatch Alarms cannot monitor raw log entries. Metric Filters convert matching log events into CloudWatch metrics that alarms can evaluate.

---

## Why did you create a custom metric?

The custom metric `FailedConsoleLoginCount` tracks failed AWS Console login attempts and provides a measurable value that CloudWatch Alarms can monitor.

---

## Why use CloudWatch Alarms?

CloudWatch Alarms continuously monitor metrics and automatically trigger actions, such as sending notifications through Amazon SNS, when thresholds are exceeded.

---

## What happens when a failed console login occurs?

1. CloudTrail records the login event.
2. CloudTrail sends the log to CloudWatch Logs.
3. The Metric Filter detects `ConsoleLogin = Failure`.
4. The custom metric `FailedConsoleLoginCount` increases.
5. CloudWatch Alarm enters the ALARM state.
6. Amazon SNS sends an email notification.
