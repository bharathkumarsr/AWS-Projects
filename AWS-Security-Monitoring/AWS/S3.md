# 🪣 Amazon S3 (Simple Storage Service)

## 📌 Overview

Amazon Simple Storage Service (Amazon S3) is a highly durable, scalable, and secure object storage service provided by AWS. It is designed to store and retrieve any amount of data from anywhere and is widely used for log storage, backups, static website hosting, disaster recovery, and data archiving.

In this project, Amazon S3 served as the **central storage layer** for AWS security logs and compliance data generated by AWS CloudTrail and AWS Config.

---

# 🎯 Objective

The objective of implementing Amazon S3 in this project was to:

- Store CloudTrail audit logs securely.
- Store AWS Config configuration snapshots and history.
- Enable long-term log retention.
- Maintain an immutable audit trail.
- Support compliance and forensic investigations.

---

# 🏗️ What We Implemented

During this project, two dedicated S3 buckets were created.

---

# 1️⃣ CloudTrail Log Bucket

## Bucket Name

```text
bharath-security-cloudtrail-logs
```

### Purpose

This bucket stores all CloudTrail log files generated from AWS account activities.

### Configuration

- Region: Mumbai (ap-south-1)
- Block Public Access: Enabled
- Versioning: Disabled
- Encryption: SSE-S3
- Used only by CloudTrail

# Security Best Practices Followed

- Separate buckets for different AWS services.
- Blocked all public access.
- Enabled server-side encryption.
- Used service-specific bucket policies.
- Stored logs in the same AWS Region.
- Restricted write access to AWS services only.

# Interview Questions

## Why did you create two S3 buckets?

Using separate buckets improves security, simplifies permission management, and follows AWS best practices by isolating CloudTrail logs from AWS Config data.

---

## Why is S3 used with CloudTrail?

CloudTrail generates audit logs that require durable, scalable storage. Amazon S3 provides highly durable storage with encryption and lifecycle management.

---

## Why does AWS Config require S3?

AWS Config continuously records resource configurations and compliance history. These snapshots need persistent storage, which is provided by Amazon S3.

---

## Why enable Block Public Access?

Security logs contain sensitive information. Enabling Block Public Access ensures these logs cannot be accidentally exposed to the internet.
